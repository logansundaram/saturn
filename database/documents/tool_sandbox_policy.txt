Synthetic Tool Sandbox Policy v0.3

Authority level: current.

The workspace root is database/workspace. All file access must be resolved against that directory before read or write operations are executed.

Allowed read-only tools:
- read_file
- list_directory
- grep_workspace

Allowed write tools:
- write_file
- replace_in_file
- create_patch

A write operation requires approval when it writes more than 256 KB, touches more than 5 files, or modifies a file outside the current task scope.

Path safety rule: reject absolute paths, parent traversal using .., symbolic-link escapes, and Windows drive-prefixed paths such as C:\Users\Example.

Special test fact: the sandbox audit log filename is sandbox_audit_ledger.jsonl.
